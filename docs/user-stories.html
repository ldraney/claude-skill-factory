MVP Stories

US-001: Batch URL Summarization

MVP
"As an operator, I submit a CSV of 100 URLs. The system queues them, runs a 'summarize' skill on each, and writes structured JSON summaries to a database table I can query."

Acceptance Criteria

  • Can upload CSV with single "url" column
  • Returns batch_id immediately
  • GET /batch/{id} shows progress (queued/processing/completed)
  • Completed batch includes structured results for each URL
  • Failed items have categorized error types
  • Results persist in Postgres for later query

Example Request

curl -X POST http://localhost:8000/batch/csv/url_summarizer \
  -F "file=@urls.csv"

# Response
{
  "batch_id": "abc-123",
  "status": "queued",
  "item_count": 100,
  "message": "CSV batch queued..."
}

US-002: Skill Discovery

MVP
"As a developer, I can list all available skills and see their input/output schemas so I know how to format my batch data."

Acceptance Criteria

  • GET /skills returns list of all registered skills
  • Each skill includes name, description, input schema, output schema
  • Schemas are JSON Schema format (from Pydantic)

US-003: Error Visibility

MVP
"As an operator, when items fail, I can see why: bad input, API error, rate limit, invalid output. This helps me fix my data or adjust my expectations."

Acceptance Criteria

  • Failed items include error_type enum
  • Failed items include human-readable error_message
  • Batch status shows completed vs failed counts
  • Can query just failed items for debugging
